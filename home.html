GENERAL STRUCTURE

Users of the system.

Student

The first user of the system is the student.A report is submitted by a student which goes to the editor, and further actions on the report are done by editor.

Editor

The most important user of the application is the editor.Once the file is uploaded, then the database of the editor is updated.The editor first asks for the confirmation of the report from the supervisor and if confirmed,further sends the report to the reviewer for review.After the reviewer approves the report,the report is published thereby generating a unique report ID.

Supervisor

The report details are recieved by the supervisor and he the confirms/rejects the report request.If the request is rejected,then the report object gets destroyed thereby.



Reviewer

The report after confirmation from the supervisor is sent to the reviewer.The reviewer checks the report for any corrections to be made and then either approves it or reuploads it after making comments on it.

About

  • PROBLEM DEFINITION

    To build a system for publishing tech reports in an institute/organisation where evaluation of reports can be done online by the reviewer and then be published on portal/journal thereby generating a unique report ID of successful publish

  • BACLGROUND

    Publishing a report has always been a problem for the publisher as well as the one who has to check it for its quality on content and standards.It would be become many more times simpler and easier if all these series of tasks can be automated through a system. Hence we came for the idea to develop a system for this work.

  • MOTIVATION

    1) To get a official record of the work done by a person (Getting timestamp)
    2) Can be used as a draft before publishing
    3) Can act as reference for author’s publications.
    4) Records can be used while searching of previous work
